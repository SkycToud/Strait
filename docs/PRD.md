# Strait - Product Requirements Document

> 東京外国語大学の学生が情報を得たいときに必ず通る要所となる

---

## 第1部：背景・課題分析

### 1.1 プロジェクトの背景

東京外国語大学の学生は、大学生活を送る上で、多くの重要な情報にアクセスする必要があります。しかし、現状では以下のような課題があります。

**情報が分散している**
学生が必要とする情報は、大学の公式ホームページ、各部局のシステム、学生会のSNS、紙の掲示板など、複数の場所に分散しています。学生は、必要な情報を得るために、これらの場所を次々と訪問しなければなりません。

**認知コストが高い**
特に、学期予定のような重要な情報がPDF形式で提供されている場合、学生は毎回PDFを開いて確認する必要があります。この「毎回確認する」という行為は、学生の認知コストを高め、情報へのアクセスを阻害します。

**情報の鮮度が不明確**
複数の情報源から同じ内容が提供されている場合、どの情報が最新なのか、学生は判断できません。

### 1.2 プロジェクトのビジョン

Straitは、これらの課題を解決するために、以下のビジョンを掲げます。

> **「東京外国語大学の学生が情報を得たいときに必ず通る要所となる」**

このビジョンは、「海峡（Strait）」という名称に込められています。学生が大学生活を送る上で、重要な情報にアクセスする際に、必ずStraitを通じてアクセスする環境を作ることが目標です。

### 1.3 プロジェクトの目的

Straitの開発を通じて、以下を実現します。

- **学生の認知コスト削減**: 分散している情報を一元化し、学生が少ないクリック数で必要な情報にたどり着ける環境を作る。
- **学生体験の向上**: 新入生がサークルを探すとき、2年生がゼミを選ぶとき、在学生が新しいことを始めたいとき、学生の人生のマイルストーンにおいて、Straitが頼りになるプラットフォームになる。
- **大学内の情報流通の改善**: 学生が必要とする情報が何かを可視化し、大学の各部局が情報提供の方法を改善するきっかけを作る。

## 第2部：ユーザーリサーチ

### 2.1 ユーザーペルソナ

Straitの主なユーザーは、以下のペルソナで代表されます。

#### ペルソナ1：新入生（春学期）
- **背景**: 大学に入学したばかりで、キャンパスの施設やサークル、学習支援制度など、多くのことを知らない状態。
- **行動**: 新入生オリエンテーション期間中に、サークルを探したり、図書館の場所を確認したり、履修登録の方法を調べたりする。
- **課題**: 情報が多すぎて、どこから始めればいいのかわからない。

#### ペルソナ2：2年生（秋学期）
- **背景**: ゼミ選びの時期に差し掛かっており、どのゼミを選ぶかについて真剣に考えている。
- **行動**: 各ゼミの教授の研究テーマを調べたり、先輩に話を聞いたり、ゼミの詳細情報を収集する。
- **課題**: ゼミの情報が教授のシラバスにしか載っていないため、実際のゼミの雰囲気や学習内容がわかりにくい。

#### ペルソナ3：在学生（通年）
- **背景**: 大学生活に慣れており、新しいことに挑戦したいと考えている。
- **行動**: インターンシップ、留学、学生プロジェクトなど、様々な機会を探している。
- **課題**: 「こんな機会があるのか」という情報自体を知らないことが多い。

### 2.2 利用シーン

Straitが活躍する主な利用シーンは、以下の通りです。

| シーン | 説明 |
|--------|------|
| **サークル探し** | 新入生は、大学に入学して間もない時期に、自分の興味に合ったサークルを探します。新歓イベントが開催されており、多くのサークルが新入生向けの情報を発信しています。Straitは、これらの情報を一元化し、新入生が効率的にサークルを探せる環境を提供します。 |
| **ゼミ選び** | 2年生は、国際社会学部は春学期、言語文化学部は秋学期にゼミ選びを行います。Straitは、教授の公式資料と、先輩学生へのインタビューを組み合わせることで、ゼミ選びに必要な情報を提供します。 |
| **新しいことを始める** | 在学生は、インターンシップ、留学、学生プロジェクトなど、様々な機会に関心を持ちます。Straitは、これらの機会に関する情報を集約し、学生が「こんな機会があるのか」という発見ができる環境を提供します。 |
| **施設利用** | 学生は、休日に図書館で勉強しようとするとき、図書館の開館時間を確認します。Straitは、施設の開館時間をリアルタイムで表示することで、学生が迷わずに施設を利用できる環境を提供します。 |

### 2.3 ユーザーが求める情報

| 利用シーン | 求める情報 | 優先度 |
|------------|------------|--------|
| サークル探し | サークルの活動内容、活動日時、新歓情報、SNS | 高 |
| ゼミ選び | ゼミの研究テーマ、教授情報、先輩の体験談 | 高 |
| 新しいことを始める | インターンシップ、留学、学生プロジェクト情報 | 中 |
| 施設利用 | 開館時間、営業状況 | 高 |
| 学期管理 | 授業開始日、試験期間、休講情報 | 高 |

## 第3部：要件定義

### 3.1 機能要件（フェーズ1）

#### 3.1.1 学期予定機能
学生が学期の重要な日程を把握できるようにするための機能です。

**要件**
- 学期予定をカレンダー形式で表示する
- 学期予定をタイムライン形式で表示する
- カレンダー形式とタイムライン形式を切り替えられる
- 学期予定をGoogleカレンダーに追加できる（iCal形式でのエクスポート/購読）

**データソース**: 大学の公式ホームページに掲載されているPDF
**データ更新頻度**: 年1回（新年度開始前）

#### 3.1.2 施設情報機能
学生が施設の開館時間や営業状況をリアルタイムで確認できるようにするための機能です。

**要件**
- 主要施設（図書館、サークル棟、食堂など）の開館時間を表示する
- 営業時間外の場合、その旨を明確に表示する
- 短縮営業や休業の場合、警告表示を出す
- リアルタイムでの更新に対応する

**データソース**: 既存の「TUFSche」プロジェクトで開発済みのJSONデータ
**データ更新頻度**: リアルタイム（既存システムに依存）

#### 3.1.3 サークル情報機能
新入生がサークルを探すときに、サークルの基本情報にアクセスできるようにするための機能です。

**要件**
- サークルの名前（日本語＋英語）を表示する
- サークルの活動内容を表示する
- サークルの公式SNS（Instagram等）へのリンクを提供する
- 約110件のサークル情報を表示する

**データソース**
- 昨年留学生向けに集めたサークル情報（Notion）
- 各サークルへのアンケート

**サークル情報の構成**
- サムネイル写真
- 概要
- 理念/指針
- 活動内容
- 活動実態
- 執行代
- 体制
- メンバー構成（人数、学年、属性）
- 活動頻度/場所
- 魅力/大変なところ
- 年間予定
- 入会までの流れ・必要事項
- 新歓情報
- 年会費
- 選考あり/選考なし
- 募集対象
- SNS情報（Instagram/X/Facebook/HP/LINE）
- 最終更新日

**データ更新頻度**: 春と秋の新歓時期に更新（年2回）

#### 3.1.4 外部リソース導線機能
大学の公式ホームページに既に掲載されている情報へのリンク集です。

**要件**
- 留学情報への公式リンクを提供する
- 奨学金情報への公式リンクを提供する
- その他の重要な大学情報へのリンクを提供する

**データソース**: 大学の公式ホームページ
**データ更新頻度**: 大学の更新に準ずる（月1回程度）

### 3.2 非機能要件

#### 3.2.1 パフォーマンス
- ページの読み込み時間は3秒以内
- スマートフォンでのスクロールはスムーズ

#### 3.2.2 ユーザビリティ
- スマートフォンでの利用を最優先（モバイルファースト）
- 直感的なナビゲーション
- 視認性の高いデザイン

#### 3.2.3 信頼性
- 情報の出典を明記する
- 最終更新日時を表示する
- 古い情報に対する警告表示

#### 3.2.4 スケーラビリティ
- 最大500名の同時利用を想定
- 将来的なデータベース移行を視野に入れた設計

### 3.3 制約条件

| 項目 | 内容 |
|------|------|
| 開発期間 | 2週間 |
| 開発チーム | プログラミング初学者2名 |
| 予算 | 0円（無料ホスティング） |
| 技術スタック | React/Next.js + Tailwind CSS + Netlify |

### 3.4 フェーズ2以降で実装する機能（スコープ外）

以下の機能は、フェーズ1では実装せず、フェーズ2以降で実装します。

- **ゼミ情報機能**: 教授の公式資料と先輩へのインタビュー情報を組み合わせたゼミ選び支援
- **授業情報検索機能**: 授業の詳細情報（シラバス、成績評価方法など）の検索
- **ユーザーアカウント機能**: ログイン、ブックマーク、個人化機能
- **混雑状況表示**: 施設の混雑状況のリアルタイム表示
